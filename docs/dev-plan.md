## 下一步开发顺序

### 已确认执行路线

本轮按《五四三二一》v0.2 策划案推进，但采用 Demo 可落地顺序：

1. 核心 Demo 闭环已经具备：扫雷推理 -> 房间移动 -> 搜刮/战斗/事件 -> 撤离/失败结算 -> 局外成长。
2. 下一步优先补“可试玩引导”和“关键表现反馈”，让评委能在 30 秒内理解左上角地图、房间门、搜索、撤离。
3. 再强化特殊房表现：怪物房压迫感、宝箱开启反馈、旅商 NPC、撤离装置、踩雷反馈。
4. 随机撤离点、出生点选择、真实物资列表后移，避免在 Demo 骨架稳定前扩大风险。

1. **失败抢救条款结算**
   - 血量归零后进入失败面板。

   - 若本局有零件，可选择抢救 1 个零件并转换为额外金币。

3. **宝箱房独立化**

### P1.5：局外系统（当前实施中）

#### 概述

局外系统为游戏提供跨局持久化进度，让每次撤离（成功/失败）都有意义。核心循环：

```
出发（选装备）→ 局内搜刮 → 撤离/失败 → 金币入账 → 购买物品/解锁天赋 → 出发...
```

#### 模块 A：MetaProgress 持久化

| 字段 | 说明 |
| --- | --- |
| `gold` | 全局金币余额（跨局持久） |
| `unlockedTalents` | 已解锁天赋 ID 列表 |
| `ownedItems` | 已购买的带入物品 ID 列表 |
| `equippedItems` | 当前装备的带入物品（最多 2 件） |
| `stats.totalRuns` | 累计出击次数 |
| `stats.totalExtractions` | 累计成功撤离次数 |
| `stats.totalGoldEarned` | 累计获得金币 |

存档方式：File API 写入 `meta_save.json`（本地 JSON）。

#### 模块 B：初始带入物品

| ID | 名称 | 效果 | 价格 | 类型 |
| --- | --- | --- | --- | --- |
| `armor` | 防护甲 | +25 最大血量 | 50g | 数值 |
| `whetstone` | 磨刀石 | +5 战斗力 | 40g | 数值 |
| `medkit` | 急救包 | 首次踩雷免疫伤害 | 60g | 机制 |
| `compass` | 罗盘 | 开局显示撤离点所在象限 | 80g | 机制 |
| `backpack` | 大背包 | 搜索奖励 +50% | 100g | 数值 |

规则：
- 物品用金币购买后永久拥有，不消耗。
- 每局最多装备 2 件。
- 局内效果在 `StartNewGame` 时应用。

#### 模块 C：基础天赋

| ID | 方向 | 名称 | 效果 | 价格 |
| --- | --- | --- | --- | --- |
| `talent_map` | 小地图 | 邻域感知 | 进入房间时高亮 8 邻域已探索状态 | 100g |
| `talent_mine` | 雷房 | 厚皮 | 雷伤降低 10 点（25→15） | 80g |
| `talent_monster` | 怪物 | 威压 | 怪物逃跑时间 +2 秒 | 80g |
| `talent_extract` | 撤离 | 抢救条款 | 信号中断时额外保留 +10 结算币 | 100g |
| `talent_event` | 事件 | 议价 | NPC 交易价格 15→20 金币/零件 | 120g |

规则：
- 天赋用金币解锁后永久生效。
- 不可重置，不可退还。

#### 模块 D：开始界面改版

当前：标题 + 简介 + 开始按钮。

改为三屏结构：
1. **主菜单**：标题 + 全局金币 + "出发" / "装备" / "天赋" 三按钮
2. **装备面板**（从主菜单进入）：展示已拥有物品，可勾选装备（最多 2 件），可花金币购买新物品
3. **天赋面板**（从主菜单进入）：5 个天赋一览，已解锁/可解锁/未达标状态

#### 模块 E：局结算接入

| 结局 | 金币计算 |
| --- | --- |
| 成功撤离 | 局内金币 + 零件×10（转换） → 全部入账 MetaProgress |
| 失败 | 局内金币（保底保留） + 可选抢救零件×10 → 入账 MetaProgress |

结算面板增加：本局获得 +Xg、当前总金币 Yg。

#### 实施文件

| 文件 | 职责 |
| --- | --- |
| `scripts/systems/MetaProgress.lua` | 持久化存档、金币增减、天赋/物品查询 |
| `scripts/main.lua` | 开始界面 UI、局结算调用 MetaProgress |
| `scripts/systems/RunInventory.lua` | 结算时调用 MetaProgress.AddGold() |

## 手感打磨记录

### 2026-05-30

问题：

- 最新连续移动用固定时间间隔触发位移，视觉上有顿感。
- 单步位移过大，导致玩家移动速度偏快。

处理：

- 移动改为按帧 `dt` 连续位移。
- 房间内移动速度调为 170px/s。
- 保留门口进入相邻格逻辑。
- 墙边提示增加冷却，避免按住方向键时反复刷消息。

后续验证重点：

- 低帧率下是否仍能稳定进门。
- 房间宽高变化时速度是否舒适。
- 按住方向键贴墙是否还有顿挫或提示干扰。

## v0.3 对齐计划

v0.3 详细进度评估与下一步计划见：

- `docs/v0.3-progress-assessment.md`

当前优先级：

1. 先保持现有最小可玩闭环稳定。
2. 将开发基准从 v0.2 切到 v0.3。
3. 优先做地图模式与生成规则：普通随机模式、评委手排模式、随机出生、随机撤离、普通模式无保底路径。

6. 背包仓库和怪物轻量走位战斗后移，避免和地图状态重构互相干扰。
