# 《灰尾回收 / 五四三二一》完整游戏扩展版数值分析文档：非重叠房型、撤离权分布与难度梯度

## 摘要

本文分析一个扫雷式撤离探索游戏的完整游戏难度结构。游戏地图由 (n\times n) 方格房间组成，玩家只能四向移动，但每进入一个房间会获得该房间周围 8 邻域内的雷房数量。已探索区域可以免费通行，因此真实探索成本不是几何行走步数，而是进入未知房间的次数。玩家通常必须先探索到撤离点，才能获得成功撤离权；因此撤离点不是普通终点，而是“停止权”的随机解锁事件。

本文采用如下正式前提：所有房型均不会重叠生成。雷房、事件房、宝箱房、怪物房、撤离点和普通房互斥。撤离点必然不是雷房，也不会与宝箱、事件、怪物重叠。由此，撤离点分布需要同时从两个角度理解：在无信息访问序列中，撤离点可按所有非起点随机格 (R=n^2-1) 建模；而对于会利用雷数信息避雷的玩家，撤离点更接近非雷池 (S=R-M) 中的随机目标。因此本文同时给出保守 (R) 模型与非雷池 (S) 参考模型。

当前 Demo 配置为：

[
10\times10,\quad 1\text{ 起点},\quad20\text{ 雷},\quad10\text{ 事件},\quad10\text{ 宝箱},\quad10\text{ 怪物},\quad2\text{ 不可见撤离点},\quad47\text{ 普通房}.
]

分析结论是：当前 Demo 的 (10\times10)、2 个不可见撤离点配置，是健康的标准偏难配置。它保留了“必须找撤离点”的搜打撤压力，同时不会像单撤离点那样产生过重的强制探索尾部。对于会利用雷数信息的玩家，因为撤离点必定非雷，真实撤离点发现时间会比全未知保守模型更早。因此当前 Demo 对推理玩家比保守模型预测更友好。

完整游戏推荐难度结构为：

[
7\times7:\quad1\text{ 可见撤离点}/1\text{ 不可见撤离点}
]

[
10\times10:\quad3/2/1\text{ 个不可见撤离点}
]

[
13\times13:\quad4/2/1\text{ 个不可见撤离点}
]

分别对应教学/入门、中图简单/标准/困难、大图普通/困难/地狱。

## 1. 问题提出

当前游戏的核心循环可以抽象为：

[
\text{进入未知房间}
\rightarrow
\text{获得雷数信息}
\rightarrow
\text{判断候选格风险}
\rightarrow
\text{积累收益与压力}
\rightarrow
\text{寻找撤离点}
\rightarrow
\text{选择撤离或继续探索}.
]

该循环不同于传统最短路问题。由于玩家不知道撤离点位置，且已探索区域可免费通行，游戏的主要消耗不是从一个点走到另一个点的几何距离，而是每次打开未知房间时承担的雷伤、压力、怪物、事件与失败损失风险。

因此，难度不应只看：

[
\mathbb E[T_{\text{exit}}],
]

即平均多久找到撤离点，而应同时看：

[
T_{90},\quad T_{95},\quad P(\text{alive at exit}),\quad HP_T,\quad Pressure_T,\quad V_T-B_T,\quad P(\Delta>0).
]

其中 (T_{90})、(T_{95}) 表示撤离权解锁尾部，(P(\text{alive at exit})) 表示活着发现撤离点概率，(HP_T) 与 (Pressure_T) 表示发现撤离点时状态，(V_T-B_T) 表示风险资产，(P(\Delta>0)) 表示发现撤离点后继续探索的倾向。

本文的目标是给出一套能够服务完整游戏的难度评价模型，并据此评估当前 Demo 和后续不同地图尺寸下的难度配置。

## 2. 基础规则与房型生成前提

地图为：

[
n\times n.
]

起点固定且安全。除起点外的随机格数量为：

[
R=n^2-1.
]

所有房型互斥生成。若雷房数量为 (M)，不可见撤离点数量为 (E)，事件、宝箱、怪物房数量分别为：

[
c_{\text{event}},\quad c_{\text{chest}},\quad c_{\text{monster}},
]

则普通房数量为：

[
N_0=R-M-E-c_{\text{event}}-c_{\text{chest}}-c_{\text{monster}}.
]

当前按比例扩展的基础配置为：

[
7\times7:\quad R=48,\quad M=10,\quad c_{\text{event}}=c_{\text{chest}}=c_{\text{monster}}=5.
]

[
10\times10:\quad R=99,\quad M=20,\quad c_{\text{event}}=c_{\text{chest}}=c_{\text{monster}}=10.
]

[
13\times13:\quad R=168,\quad M=34,\quad c_{\text{event}}=c_{\text{chest}}=c_{\text{monster}}=17.
]

原始雷率为：

[
p_{\text{raw}}=\frac{M}{R}.
]

因此：

[
7\times7:\quad \frac{10}{48}\approx20.8%.
]

[
10\times10:\quad \frac{20}{99}\approx20.2%.
]

[
13\times13:\quad \frac{34}{168}\approx20.2%.
]

三档地图的原始雷率基本一致。跨地图难度差异主要来自地图面积、撤离点数量、撤离权尾部、内容曝光、信息复杂度和玩家策略，而不是单纯雷密度。

当前 Demo 使用 (10\times10)、2 个不可见撤离点，因此普通房数量为：

[
99-20-10-10-10-2=47.
]

即：

[
\boxed{
1\text{ 起点}+20\text{ 雷}+10\text{ 事件}+10\text{ 宝箱}+10\text{ 怪物}+2\text{ 不可见撤离点}+47\text{ 普通房}.
}
]

## 3. 移动、信息与真实探索成本

玩家移动采用四邻接规则。若两个格子为：

[
(r_1,c_1),\quad(r_2,c_2),
]

则已知目标时的最短距离是曼哈顿距离：

[
d=|r_1-r_2|+|c_1-c_2|.
]

但游戏初期撤离点未知，雷房未知，玩家也不能直接选择最短路。因此几何距离不是核心难度。

玩家进入非雷房 (v) 后可看到其周围 8 邻域内的雷房数量：

[
C(v)=\sum_{u\in N_8(v)}\mathbf 1_{{u\text{ 为雷}}}.
]

当前没有传统扫雷的空白格自动展开规则。即使数字为 0，也只是表示周围 8 格无雷，不会自动展开更大区域。

设玩家当前已知信息为：

[
\mathcal I_t.
]

候选未知格集合为：

[
A_t.
]

对任意候选格 (a)，其后验雷率为：

[
p_a=P(a\text{ 是雷}\mid \mathcal I_t).
]

原始雷率 (p_{\text{raw}}) 只是地图密度。真正决定玩家生存压力的是策略 (\pi) 下的有效踩雷率：

[
q_\pi=
\mathbb E[
P(a_t\text{ 是雷}\mid \mathcal I_t,\pi)
].
]

因此，完整游戏中的难度评估应区分至少四类策略：

[
\pi_{\text{blind}}
]

盲走玩家；

[
\pi_{\text{minrisk}}
]

选择当前最低雷率格的保守玩家；

[
\pi_{\text{human-noisy}}
]

带有误判率的人类玩家；

[
\pi_{\text{bayes-optimal}}
]

非全知但贝叶斯理性的最优玩家。

本文后续用 (q=2%) 与 (q=5%) 做设计初筛，但这只是玩家模型假设。真实 (q_{\text{actual}}) 必须通过模拟或玩家日志校准。

## 4. 撤离点作为停止权

撤离点不是普通房间，而是成功撤离权的解锁事件。

发现撤离点前，玩家无法主动成功撤离，只能继续探索或失败。发现撤离点后，玩家才拥有停止权，可以选择立即撤离或继续探索。

设不可见撤离点数量为 (E)。若玩家在发现撤离点之前的访问序列不依赖撤离点位置，则 (E) 个撤离点在 (R) 个随机格中的访问名次相当于从 (1,\dots,R) 中无放回抽取 (E) 个位置。

首次发现撤离点时间为：

[
T_E=\min(T_1,\dots,T_E).
]

其分布为：

[
P(T_E>t)=\frac{\binom{R-t}{E}}{\binom{R}{E}}.
]

因此：

[
\boxed{
P(T_E\le t)=1-\frac{\binom{R-t}{E}}{\binom{R}{E}}.
}
]

期望为：

[
\boxed{
\mathbb E[T_E]=\frac{R+1}{E+1}=\frac{n^2}{E+1}.
}
]

撤离点数量增加不仅降低平均值，还降低每一个时间点之后仍未找到撤离点的概率。换言之，更多撤离点对首次发现时间形成一阶随机支配：

[
P(T_{E+1}\le t)\ge P(T_E\le t).
]

因此，撤离点数量控制的不只是平均撤离时间，而是整个强制探索阶段的尾部风险。

## 5. 房型不重叠后的非雷池参考模型

由于所有房型不重叠，撤离点必然不是雷房。设非雷随机格数量为：

[
S=R-M.
]

对于完全无信息访问模型，撤离点仍可用 (R) 模型估计。但对于会利用雷数信息的玩家，探索路径会倾向于低雷率格，撤离点也更可能在玩家倾向访问的非雷区域中被较早发现。

因此，本文同时给出两个口径：

[
\boxed{
R\text{ 模型：全未知格保守基准。}
}
]

[
\boxed{
S=R-M\text{ 模型：强避雷 / 非雷池参考。}
}
]

在非雷池参考模型中：

[
P(T_E^{\text{safe}}\le t)
=========================

1-\frac{\binom{S-t}{E}}{\binom{S}{E}}.
]

[
\boxed{
\mathbb E[T_E^{\text{safe}}]=\frac{S+1}{E+1}=\frac{R-M+1}{E+1}.
}
]

真实玩家表现介于两者之间，具体位置由 (q_\pi) 决定。

因此：

[
\frac{R-M+1}{E+1}
\le
\mathbb E[T_E^{\text{actual}}]
\le
\frac{R+1}{E+1}.
]

对当前 Demo：

[
R=99,\quad M=20,\quad S=79,\quad E=2.
]

保守 (R) 模型：

[
\mathbb E[T_E]=\frac{100}{3}\approx33.3.
]

非雷池参考模型：

[
\mathbb E[T_E^{\text{safe}}]=\frac{80}{3}\approx26.7.
]

这说明当前 Demo 对会利用雷数信息的玩家比保守模型更友好。

## 6. 候选格撤离概率与雷率的联动

房型不重叠后，候选格为撤离点的概率与其雷率不再独立。若：

[
p_a=P(a\text{ 是雷}\mid \mathcal I),
]

则：

[
a\text{ 是雷}\Rightarrow a\text{ 不可能是撤离点}.
]

设剩余未发现撤离点数量为 (E_{\text{rem}})，未知格集合为 (U)，则一个合理的一阶近似为：

[
\boxed{
\lambda_a
=========

P(a\text{ 是撤离点}\mid\mathcal I)
\approx
E_{\text{rem}}
\cdot
\frac{1-p_a}{\sum_{u\in U}(1-p_u)}.
}
]

因此：

[
p_a=1\Rightarrow \lambda_a=0.
]

[
p_a=0\Rightarrow \lambda_a\text{ 相对更高}.
]

这意味着低雷率格有双重价值：

[
\boxed{
更安全，并且更可能解锁撤离权。
}
]

这也是为什么会推理玩家的撤离权发现时间会早于无信息 (R) 模型。

## 7. 强制探索阶段的价值函数

设游戏状态为：

[
S=(\mathcal I,HP,Pressure,Inventory,ExitFound).
]

当：

[
ExitFound=0,
]

玩家处于强制探索阶段，没有主动成功撤离选项。

当：

[
ExitFound=1,
]

玩家拥有停止权，可以选择撤离或继续探索。

所有房型互斥，因此候选格 (a) 只可能属于以下类型之一：

[
\tau\in
{\text{Mine},\text{Exit},\text{Event},\text{Chest},\text{Monster},\text{Normal}}.
]

强制探索阶段探索 (a) 的价值可写为：

[
Q_0(S,a)
========

P_{\text{death}}(a)B(S)
+
\sum_{\tau}
P(a=\tau,\text{alive}\mid \mathcal I)
[
R_\tau(S,a)+F_{E'}(S'_\tau)
].
]

其中：

[
E'=1
\quad\text{当且仅当}\quad \tau=\text{Exit}.
]

若进入后死亡，则价值为失败保底：

[
B(S).
]

若进入后存活且该格为撤离点，则进入可停止阶段：

[
F_1(S).
]

若进入后存活但不是撤离点，则仍处于强制探索阶段：

[
F_0(S).
]

当撤离点已发现后，价值函数变为：

[
F_1(S)=\max{\text{Extract}(S),\text{Explore}(S)}.
]

这个形式比单步边际式更完整。单步边际式仍有用，但只适用于撤离点已发现后的局部决策。

## 8. 生存压力模型

设玩家有效踩雷率为：

[
q.
]

设怪物遭遇并处理率为：

[
m.
]

平均每探索一个未知格的压力增长为：

[
\alpha=2+10q+5m.
]

协议 1 触发时间近似为：

[
\boxed{
\tau=\frac{80}{\alpha}.
}
]

若只考虑雷伤与协议 1 后的压力扣血，探索到第 (T) 个未知格时的期望生命损失为：

[
L(T)=30qT+2\max(0,T-\tau).
]

若怪物有稳定 HP 伤害，应加入：

[
D_m mT.
]

若事件有稳定负面 HP 伤害，应加入：

[
D_e eT.
]

则可扩展为：

[
L(T)=30qT+D_m mT+D_e eT+2\max(0,T-\tau).
]

本文主表采用简化版，把怪物先计入压力，而不计入稳定 HP 伤害。

令临界探索名次为 (T_c)，满足：

[
L(T_c)=100.
]

当 (T_c>\tau) 时：

[
30qT_c+2(T_c-\tau)=100.
]

因此：

[
\boxed{
T_c=\frac{100+2\tau}{30q+2}.
}
]

活着发现撤离点的一阶解析近似为：

[
\boxed{
P(\text{alive at exit})\approx P(T_E\le T_c).
}
]

这个值是解析初筛，不是严格生存率。更严格的生存概率应写为：

[
P(\text{alive at exit})
=======================

\sum_t P(T_E=t)S(t),
]

其中：

[
S(t)=P(\text{第 }t\text{ 格时仍存活}).
]

最终应通过蒙特卡洛模拟和真实玩家数据校准。

## 9. 当前估算参数

本文采用两组设计假设：

普通推理玩家：

[
q=2%.
]

弱推理或高失误玩家：

[
q=5%.
]

怪物遭遇并处理率暂取：

[
m=10%.
]

当 (q=2%) 时：

[
\alpha=2+10(0.02)+5(0.10)=2.7.
]

[
\tau=\frac{80}{2.7}\approx29.6.
]

[
T_c=\frac{100+2(29.6)}{30(0.02)+2}
==================================

\frac{159.2}{2.6}
\approx61.3.
]

本文表格按保守整数口径取：

[
T_c=61.
]

当 (q=5%) 时：

[
\alpha=2+10(0.05)+5(0.10)=3.0.
]

[
\tau=\frac{80}{3.0}\approx26.7.
]

[
T_c=\frac{100+2(26.7)}{30(0.05)+2}
==================================

\frac{153.4}{3.5}
\approx43.8.
]

本文表格按保守整数口径取：

[
T_c=43.
]

因此，后续 (P_{\text{alive}}) 均为一阶解析近似，且存在取整误差。

## 10. 内容曝光模型

若某类内容房数量为 (c)，不可见撤离点数量为 (E)，则在首次发现撤离点之前，该类内容房的无死亡基准期望出现数为：

[
\boxed{
\mathbb E[C_{\text{before}}]=\frac{c}{E+1}.
}
]

该类内容至少出现一次的概率为：

[
\boxed{
P(C_{\text{before}}\ge1)=\frac{c}{c+E}.
}
]

这只是撤离前、无死亡、无策略偏置的基准曝光。由于事件、宝箱、怪物和撤离点均为非雷房，会避雷玩家更倾向访问非雷房，因此实际内容曝光可能偏离该基准。最终应通过策略模拟得到：

[
C_{\text{before}}^\pi,\quad C_{\text{after}}^\pi.
]

完整局内内容曝光应写为：

[
C_{\text{total}}
================

C_{\text{before exit}}
+
C_{\text{after exit}}.
]

若玩家发现撤离点后平均继续探索 (K_{\text{after}}) 个未知格，则可近似估计：

[
C_{\text{total}}
\approx
\frac{c}{E+1}
+
K_{\text{after}}\cdot \frac{c}{R}.
]

这解释了为什么撤离点多并不一定导致总内容体验不足：玩家更早获得退路后，往往更愿意继续探索。

## 11. 发现撤离点后的继续探索

当撤离点已发现后，玩家可以选择立即撤离。此时继续探索一格的边际期望为：

[
\Delta=(1-f)Y-f(V-B).
]

其中：

[
Y
]

是存活时新增收益；

[
f
]

是本次探索导致失败的概率；

[
V
]

是当前成功撤离价值；

[
B
]

是失败保底价值。

当：

[
\Delta>0,
]

继续探索在期望上划算。

当：

[
\Delta<0,
]

立即撤离更优。

由于失败时待结算币丢失、物资只能保留 1 件，(B) 不应写成固定比例，而应是状态函数：

[
B=B(S).
]

风险资产为：

[
A=V-B(S).
]

撤离点越多，玩家越早获得停止权。一般会导致：

[
HP_T\uparrow,\quad Pressure_T\downarrow,\quad V_T\downarrow.
]

因此简单难度下，玩家更可能找到撤离点后继续探索；困难难度下，撤离点更像真正的止损节点。

[
7\times7:
\quad
\text{简单}=1\text{ 可见撤离点},
\quad
\text{普通}=1\text{ 不可见撤离点}.
]

[
10\times10:
\quad
\text{简单}=3\text{ 不可见撤离点},
\quad
\text{标准 / Demo}=2\text{ 不可见撤离点},
\quad
\text{困难}=1\text{ 不可见撤离点}.
]

[
13\times13:
\quad
\text{普通}=4\text{ 不可见撤离点},
\quad
\text{困难}=2\text{ 不可见撤离点},
\quad
\text{地狱}=1\text{ 不可见撤离点}.
]

可见撤离点需要进一步定义。本文建议：

[
\boxed{
可见撤离点表示位置可见，但路径仍需按规则探索。

## 15. 盲走玩家的多撤离点雷伤上界

对完全盲走玩家，仅考虑雷伤且不考虑压力扣血时，可以只看雷房和撤离点的相对顺序。

设雷房数量为 (M)，不可见撤离点数量为 (E)。玩家第 4 次踩雷失败，因此活着发现撤离点的条件是第一个撤离点出现前雷房数 (K\le3)。

概率为：

[
\boxed{
P(K\le3)
========

\sum_{k=0}^{3}
\frac{\binom{M+E-k-1}{E-1}}{\binom{M+E}{E}}.
}
]

当前 Demo：

[
M=20,\quad E=2.
]

只考虑雷伤时，盲走玩家活着发现撤离点的上界约为：

[
33.8%.
]

若单撤离点：

[
E=1,
]

则为：

[
\frac4{21}\approx19.0%.
]

若三撤离点：

[
E=3,
]

约为：

[
45.3%.
]

加入压力扣血后，真实盲走生存率会低于这些数值。该结果说明 2 个撤离点比 1 个撤离点更适合作为 Demo 标准，但盲走玩家仍明显劣于会推理玩家。

## 16. 分难度评价

### 16.1 (7\times7) 简单：1 个可见撤离点

该配置承担教学任务。可见撤离点位置可见，玩家较快理解撤离、成功结算和继续探索。为了避免教学局早期必猜，建议提供起点周边保护、低风险起步或弱提示。

该难度目标不是制造完整压力，而是建立规则理解。

结论：

[
\boxed{
7\times7\text{ 简单适合作为教学模式。}
}
]

### 16.2 (7\times7) 普通：1 个不可见撤离点

保守模型下平均 24.5 格发现撤离点，非雷池参考下约 19.5 格。地图小，强制探索尾部可控。

结论：

[
\boxed{
7\times7\text{ 普通适合作为入门找出口模式。}
}
]

### 16.3 (10\times10) 简单：3 个不可见撤离点

保守模型下平均 25 格发现撤离点，非雷池参考下约 20 格。它保留找撤离点体验，但明显降低强制探索尾部风险。

结论：

[
\boxed{
10\times10\text{ 简单设为 3 个不可见撤离点是合理的。}
}
]

### 16.4 (10\times10) 标准 / Demo：2 个不可见撤离点

当前 Demo 即此配置。保守模型下平均 33.3 格发现撤离点，非雷池参考下约 26.7 格。普通推理玩家的活着发现撤离点概率处于健康区间，弱推理玩家仍会感到明显压力。

结论：

[
\boxed{
当前 Demo 是健康的标准偏难配置。}
]

它要求玩家理解雷数，但不会像单撤离点那样过度依赖撤离点晚不晚出现。

### 16.5 (10\times10) 困难：1 个不可见撤离点

保守模型下平均 50 格发现撤离点，非雷池参考下约 40 格。该档的困难来自强制探索阶段更长、撤离点尾部更重。

结论：

[
\boxed{
10\times10\text{ 单不可见撤离点适合作为困难 / 机密配置。}
}
]

建议配套更高收益、扫描针、首雷减伤、撤离方向弱提示或可带入道具。

### 16.6 (13\times13) 普通：4 个不可见撤离点

保守模型下平均 33.8 格发现撤离点，非雷池参考下约 27 格。它与 (10\times10) 标准在撤离节奏上接近，但地图更大、信息更多、决策分支更多，因此仍然是更复杂的大图普通。

结论：

[
\boxed{
13\times13\text{ 普通设为 4 个不可见撤离点非常合理。}
}
]

### 16.7 (13\times13) 困难：2 个不可见撤离点

保守模型下平均 56.3 格发现撤离点，非雷池参考下约 45 格。该档是真正的大图困难。

结论：

[
\boxed{
13\times13\text{ 双不可见撤离点适合作为困难。}
}
]

该档应有更高奖励或更强道具价值，否则会被部分玩家理解为“出口太晚”。

### 16.8 (13\times13) 地狱：1 个不可见撤离点

保守模型下平均 84.5 格发现撤离点，非雷池参考下约 67.5 格。即使考虑撤离点非雷，该档仍有很重的强制探索尾部。

结论：

[
\boxed{
13\times13\text{ 单不可见撤离点应定位为地狱 / 绝密 / 高失败率挑战。}
}
]

该档必须配高收益、高成就感和明确风险提示。

## 19. 结论

本文从当前正式前提出发，建立了完整游戏扩展版的难度分析模型。核心结论如下。

第一，所有房型不重叠生成后，当前 Demo 普通房数量应为 47，而不是 48。

第二，撤离点必定非雷，因此撤离点发现时间应同时用保守 (R) 模型和非雷池 (S=R-M) 参考模型理解。真实玩家表现介于二者之间，具体位置由 (q_\pi) 决定。

第三，候选格为撤离点的概率应与雷率联动：

[
\lambda_a\approx
E_{\text{rem}}
\cdot
\frac{1-p_a}{\sum_{u\in U}(1-p_u)}.
]

低雷率格不仅更安全，也更可能解锁撤离权。

第四，当前 (10\times10)、20 雷、10 事件、10 宝箱、10 怪物、2 个不可见撤离点的 Demo 配置仍然合理，并且对会利用雷数信息的玩家比保守模型预测更友好。

第五，推荐完整游戏难度结构为：

[
\boxed{
7\times7:\quad1可见/1不可见
}
]

[
\boxed{
10\times10:\quad3/2/1不可见
}
]

[
\boxed{
13\times13:\quad4/2/1不可见
}
]

其中 (10\times10)、2 不可见撤离点可作为当前 Demo 标准模式；(10\times10)、3 不可见撤离点适合新手友好版本；(13\times13)、1 不可见撤离点应定位为地狱 / 绝密挑战。

第六，解析模型只能作为初筛。后续必须通过策略模拟和真实玩家数据校准 (q_\pi)、(P(\text{alive}))、(HP_T)、(Pressure_T)、(P(\Delta>0))、最终收益和失败损失。

# 附录 B：解析计算程序

以下程序用于计算保守 (R) 模型与非雷池 (S) 参考模型下的撤离点期望、90% 分位、生存近似与内容曝光。它是解析初筛程序，不是完整蒙特卡洛模拟。

```python
from math import comb

def exit_cdf(N, E, t):
    if E <= 0:
        return 0.0
    if t <= 0:
        return 0.0
    if t >= N:
        return 1.0
    if N - t < E:
        return 1.0
    return 1 - comb(N - t, E) / comb(N, E)

def exit_expectation(N, E):
    return (N + 1) / (E + 1)

def exit_quantile(N, E, p=0.9):
    for t in range(1, N + 1):
        if exit_cdf(N, E, t) >= p:
            return t
    return N

def pressure_alpha(q, m):
    return 2 + 10 * q + 5 * m

def protocol_trigger_time(q, m):
    return 80 / pressure_alpha(q, m)

def critical_time(q, m):
    tau = protocol_trigger_time(q, m)
    return (100 + 2 * tau) / (30 * q + 2)

def alive_probability(N, E, Tc):
    return exit_cdf(N, E, int(Tc))

def content_before_expectation(c, E):
    return c / (E + 1)

def content_seen_once_probability(c, E):
    return c / (c + E)

configs = [
    ("7x7 普通", 48, 10, 1, 5),
    ("10x10 简单", 99, 20, 3, 10),
    ("10x10 标准/Demo", 99, 20, 2, 10),
    ("10x10 困难", 99, 20, 1, 10),
    ("13x13 普通", 168, 34, 4, 17),
    ("13x13 困难", 168, 34, 2, 17),
    ("13x13 地狱", 168, 34, 1, 17),
]

q_normal = 0.02
q_weak = 0.05
m = 0.10

Tc_normal = int(critical_time(q_normal, m))
Tc_weak = int(critical_time(q_weak, m))

print("Tc normal:", Tc_normal)
print("Tc weak:", Tc_weak)

for name, R, M, E, c in configs:
    S = R - M

    result = {
        "config": name,
        "R": R,
        "M": M,
        "S": S,
        "E": E,
        "E[T] R": round(exit_expectation(R, E), 2),
        "E[T] S": round(exit_expectation(S, E), 2),
        "T90 R": exit_quantile(R, E, 0.9),
        "T90 S": exit_quantile(S, E, 0.9),
        "Alive q=2% R": round(alive_probability(R, E, Tc_normal), 3),
        "Alive q=2% S": round(alive_probability(S, E, Tc_normal), 3),
        "Alive q=5% R": round(alive_probability(R, E, Tc_weak), 3),
        "Alive q=5% S": round(alive_probability(S, E, Tc_weak), 3),
        "Content before": round(content_before_expectation(c, E), 2),
        "Content once": round(content_seen_once_probability(c, E), 3),
    }

    print(result)
```
