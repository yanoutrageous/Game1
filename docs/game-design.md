## 1. 游戏概述

### 1.1 一句话概括

玩家在一张基于经典扫雷规则生成的地牢地图中探索。每个格子对应一个 2D 俯视角房间，玩家通过小地图上的扫雷数字判断雷房位置，探索宝箱、怪物、事件与撤离点，在倒计时压力下搜刮物资并尝试成功撤离。

### 1.2 核心玩法定位

本游戏的核心不是直接操作扫雷棋盘，而是：

> 将扫雷数字作为地牢探索中的风险情报系统。

玩家在房间中行动，小地图提供信息。玩家需要根据数字推理雷房位置，同时判断怪物房、宝箱房、事件房、撤离房带来的收益与风险。

### 1.3 核心体验

1. 扫雷推理：通过小地图数字判断附近雷房。
2. 地牢探索：在俯视角房间中移动、搜索、交互。
3. 风险收益：雷房是确定损失，怪物房可能是收益，也可能是损失。
4. 搜打撤压力：玩家不能随时成功撤离，必须找到撤离房。
5. 失败取舍：撤离失败后，玩家需要从初始物资和局内物资中选择部分保留。
6. 五四三二一倒计时：通过倒计时协议强化主题与局内紧张感。

### 1.4 当前版本设计主轴

> 经典矩形扫雷地图 + 2D 俯视角房间探索 + 雷房确定扣血 + 怪物房战斗力检定 + 宝箱/事件收益 + 随机撤离点 + 信号中断抢救条款 + 五四三二一倒计时。

## 3. 游戏主题：“五四三二一”

### 3.1 主题定位

“五四三二一”在本游戏中表现为：

> 撤离协议倒计时。

它代表当前区域的危险等级、封锁进程或撤离压力。

### 3.2 倒计时基础规则

倒计时从 5 逐步下降至 1。推荐不使用真实时间倒计时，而是使用“探索进度”推进。

理由：

- 真实时间倒计时会干扰扫雷推理。
- 玩家需要时间观察小地图。
- 探索房间数量更符合“越深入越危险”的撤离体验。

当前 Demo 实现：按已探索唯一房间数推进协议等级。

| 协议等级 | 当前 Demo 触发条件 | 后续可调方向 |
| --- | --- | --- |
| 5 | 初始阶段 | 无负面 |
| 4 | 探索 4 个房间 | UI / 音效变化 |
| 3 | 探索 8 个房间 | 轻微怪物相关 debuff |
| 2 | 探索 12 个房间 | 轻微雷房相关 debuff |
| 1 | 探索 16 个房间 | 低等级警报与结算压力 |

注意：

> 倒计时降至 1 不应直接导致失败。核心仍然是扫雷推理与撤离探索，不是纯倒计时逃生。

### 3.3 美术与 UI 表现

【美术需补充】协议数字 5/4/3/2/1 的显示样式、阶段变化动画、阶段颜色变化、低等级警报表现，以及是否联动小地图边框、房间灯光、环境音。

【文案待定】协议名称可选：撤离协议、封锁协议、五四三二一协议。

## 4. 地图系统

### 4.1 地图类型

Demo 采用经典矩形扫雷地图。蜂窝地图、特殊规则地图、多层地图作为后续扩展。

### 4.2 难度映射

| 扫雷难度 | 游戏内难度名称 | 状态 |
| --- | --- | --- |
| 简单 | 普通 | Demo 优先 |
| 中档 | 机密 | 后续扩展 |
| 困难 | 绝密 | 后续扩展 |

难度影响地图尺寸、雷房数量、怪物强度、奖励价值、撤离点数量、抢救条款数量、倒计时惩罚强度和高稀有度物资概率。

### 4.3 地图尺寸与雷数

当前策划倾向参考经典扫雷基础配置，但现有程序原型使用 15x15 并围绕“中心出生 -> 四角撤离”结构设计。

【与现有程序原型冲突】待讨论 Demo 是否改为 9x9、11x11、13x13，或保留 15x15 以支持四角撤离空间。

### 4.4 出生点规则

策划目标：

1. 玩家先选择出生点。
2. 系统根据出生点生成地图。
3. 出生点本身不得为雷房。
4. 出生点周围不保证安全。
5. 极端开局允许存在。
6. 该设计提升“第一次伤害免疫”等带入物品价值。

【与现有程序原型冲突】当前实现为中心固定出生，中心出生点永远不是地雷。Demo 可先保留中心出生。

### 4.5 撤离点规则

策划目标：

1. 玩家不能主动随时撤离。
2. 退出当局视为撤离失败。
3. 玩家必须探索并找到撤离房。
4. 撤离点随机生成。
5. 撤离点可重复进入。
6. 使用撤离点一次后，本局强制结束。
7. 撤离点可以被雷区包围。
8. 撤离点距离出生点不做强限制。

【与现有程序原型冲突】当前实现为四角固定撤离点，四个角落始终可见且永不为雷。Demo 可先保留四角撤离点，后续再切随机撤离房。

### 4.6 地图连通与保底路径

策划倾向不强制保证玩家到撤离点的安全路径，以保留扫雷的不确定性和搜打撤压力。

【与现有程序原型冲突】当前实现会保留从中心到四角的安全路径。为评委试玩稳定性，Demo 阶段建议暂时保留。

### 4.7 邻接规则

| 层级 | 规则 |
| --- | --- |
| 房间移动 | 上、下、左、右四方向门 |
| 扫雷计算 | 当前格周围 8 格统计雷房数量 |

玩家只能通过四个门移动，但斜向格仍参与数字计算。

## 5. 小地图系统

小地图是核心玩法系统，不是辅助 UI。它负责显示探索状态、当前位置、雷数、房间类型、标记、放大查看和已探索房间传送。

### 5.1 基础显示规则

| 状态 | 显示 |
| --- | --- |
| 未探索房间 | 未知格 / 盖住格 |
| 当前房间 | 玩家图标 / 高亮框 |
| 普通房 | 周围 8 格雷数 |
| 雷房 | 触发后显示雷图标 |
| 怪物房 | 探索后显示怪物图标 + 雷数 |
| 宝箱房 | 探索后显示宝箱图标 + 雷数 |
| 事件房 | 探索后显示事件图标 + 雷数 |
| 撤离房 | 探索后显示撤离图标 + 雷数 |
| 标记格 | 显示玩家标记图标 |

重要规则：

> 所有非雷房间在探索后都应显示周围 8 格雷数，同时显示自身类型图标。

### 5.2 放大地图

放大地图可以查看数字、旗标、当前位置、房间类型，并支持已探索房间传送和标记/取消标记。

当前实现：支持放大、插旗、已访问安全格回传。特殊房图标层级待补。

### 5.3 标记功能

推荐折中方向：

> Demo 默认开放基础标记；局外天赋解锁高级标记、多色标记、自动记录等功能。

### 5.4 地图传送

规则：

- 只能传送到已探索房间。
- 未探索房间不能传送。
- 传送不揭示新房间。
- 传送不触发新事件。
- 怪物追击/逃跑阶段不能传送。
- 战斗结算中不能传送。
- 撤离确认中不能传送。

当前实现：只传送到已访问安全格。已触发雷房是否可传送待讨论。

### 5.5 0 格连锁展开

当前实现包含 0 格连锁展开。推荐后续区分“信息揭示”和“玩家亲自进入”：0 展开只揭示小地图信息，不自动触发宝箱、怪物、事件或撤离。

## 6. 房间系统

### 6.1 房间类型

| 房间类型 | 作用 | Demo 状态 |
| --- | --- | --- |
| 普通房 | 提供雷数信息 | 已实现 |
| 雷房 | 确定扣血 | 已实现 |
| 怪物房 | 战斗力检定，可能转化为收益 | 已实现独立房型 |
| 宝箱房 | 获得 BUFF 或物资 | 已实现独立房型 |
| 事件房 | NPC、交易、置换等 | 已实现旅商交易 |
| 撤离房 | 成功撤离 | 四角固定撤离点 |

所有房间类型原则上互不重叠。

### 6.2 雷房

雷房是确定负收益房间，进入后必定扣血。

规则：

1. 雷房计入扫雷数字。
2. 怪物房不计入雷数。
3. 宝箱房、事件房、撤离房不计入雷数。
4. 玩家进入雷房后扣血。
5. 雷房只触发一次。
6. 已触发雷房不再重复扣血。
7. 已触发雷房可作为通路。
8. 普通难度下雷房不应直接造成必死。

当前 Demo 实现：踩雷扣血，血量归零才失败；同一雷房重复进入不再扣血。

### 6.3 怪物房

策划目标：怪物房不计入雷数，是独立收益/风险房间。玩家战斗力足够时怪物房可以转化为物资来源。

当前 Demo 实现：怪物房已作为独立 `roomType`，不计入雷数。进入后显示怪物强度与玩家战斗力，并通过 VS 演出结算；威压天赋会提供额外逃跑窗口。

### 6.4 怪物战斗结算

| 情况 | 结果 |
| --- | --- |
| 玩家战斗力 >= 怪物强度 | 击败怪物，获得掉落 |
| 玩家战斗力 < 怪物强度 | 根据差值或公式扣血，随后结算 |
| 逃跑失败 | 视为进行一次战斗 |

当前倾向：避免怪物永久堵路。即使玩家付出血量代价，也尽量完成一次战斗结算。

### 6.5 宝箱房

宝箱房是正收益房间，提供局内 BUFF 或可带出物资。当前 Demo 已作为独立 `roomType`，进入后可按 F 开启宝箱并获得更高收益。

### 6.6 事件房

当前 Demo 实现为旅商 NPC：玩家可将 1 个零件提前转换为金币，议价天赋可提升兑换价格。金币作为安全资产，失败时自动保留。

### 6.7 撤离房

策划目标：撤离房随机生成、需要探索发现、进入后选择是否撤离、成功带出全部局内物资。

当前 Demo 实现：四角固定撤离点，始终可见。后续切换为随机撤离点前，需要先做撤离点发现与图标揭示。

## 9. 撤离与失败结算

### 9.1 成功撤离

触发条件：玩家进入撤离房，并选择撤离。

结果：

1. 本局强制结束。
2. 初始带入物品不消耗。
3. 局内获得的可带出物资全部带出。
4. 可带出物资结算为金币。
5. 已转化金币保留。
6. 局内 BUFF 失效。

当前 Demo 实现：四角撤离点按 E 撤离，展示金币、零件和搜索房间数。

### 9.2 撤离失败

触发条件：血量归零、玩家主动退出当局、其他失败条件。

结果：

1. 不能带出全部物资。
2. 初始带入物品与局内可带出物资共同进入保底选择。
3. 玩家只能选择有限数量物品保留。
4. 未选择保留的物品被消耗。
5. 已转化金币保留。
6. 全局金币不受影响。
7. 局内 BUFF 全部失效。

当前 Demo 已实现失败抢救条款：已锁定收益自动保留，待结算收益默认丢失；若本局有回收物，可按条款抢救 1 件。

## 10. 局外成长系统

局外成长不应只做单纯数值提升，机制型天赋为主，数值提升为辅。

| 天赋方向 | 示例 |
| --- | --- |
| 小地图类 | 解锁标记、多色标记、自动记录、高亮 8 邻域 |
| 雷房类 | 首次雷伤免疫、雷伤降低、低血保命 |
| 怪物类 | 逃跑时间增加、怪物掉落提升 |
| 撤离类 | 失败额外保留物品、撤离点感知 |
| 事件类 | NPC 交易价格提高、交易次数增加 |

局外天赋系统作为 P1/P2，不应影响 P0 可玩闭环。

## 18. 当前策划结论

当前策划方向保持为：

> 扫雷情报驱动的 2D 俯视角撤离地牢。

核心设计仍然是：

1. 小地图使用经典扫雷 8 向数字。
2. 玩家在 2D 房间中四方向移动。
3. 雷房是确定扣血。
4. 怪物房不是雷房，而是战斗力收益/风险检定。
5. 宝箱房和事件房提供收益。
6. 玩家必须找到撤离房才能成功带出物资。
7. 撤离失败后通过保底选择降低挫败。
8. 五四三二一倒计时承担主题表达与轻微压力。

现有程序原型可以视为已经完成的最小可玩骨架，本策划案作为后续逐步对齐的目标设计版本。
